## Primary: Spec Compliance

1. **Requirements compliance**: Verify each referenced requirement is fully implemented: 1.1, 2.4. For each one, trace from the requirement to the specific code that fulfills it. 3. **Success criteria**: The success criteria are: All listed cases pass; messages emit verbatim. For each criterion, find concrete evidence in the code that it is met. If a criterion is ambiguous, interpret it strictly. Flag criteria that would fail under edge cases or with unexpected input.
4. **File scope**: Compare the files actually modified/created against what the task specifies. Flag any files touched that are outside the task's scope (scope creep).
